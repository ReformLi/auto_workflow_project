# 脚本工作流可视化编辑器

基于PyQt5和NodeGraphQt构建的可视化脚本工作流编辑器。

## 功能特性

- **可视化节点编辑器**: 支持拖拽式节点创建和连接
- **丰富的节点类型**: 包含开始、结束、数据处理、条件判断、循环、文件操作、网络请求、数据库等节点
- **完整的用户界面**: 包含菜单栏、工具栏、节点库、编辑器和日志窗口
- **工作流管理**: 支持新建、打开、保存工作流文件
- **实时日志**: 底部可滚动的日志显示区域
- **工作流验证**: 内置工作流有效性验证功能

## 系统要求

- Python 3.7+
- PyQt5 5.15.10
- NodeGraphQt 0.6.44

## 安装依赖

```bash
pip install -r requirements.txt
```

## 项目结构

```
window_automation_workflow/
├── app                        # 应用入口与全局配置
├── core/                       # 核心业务层（逻辑中枢）
├── nodes/                     # 节点模块
│   └── base_nodes.py          # 基础节点定义 (5KB)
├── ui/                         # 用户界面层（视图组件）
├── automation/                 # 窗口自动化引擎（底层执行）
├── resources/                  # 静态资源
└── tests/                      # 测试模块
```

## 界面布局

- **菜单栏**: 包含文件、编辑、运行三个菜单
- **工具栏**: 提供新建、保存、打开、清空等常用功能的快速访问
- **左侧面板**: 节点库，显示可用的节点类型（占1/6宽度）
- **右侧面板**: 节点图编辑器，主要的可视化编辑区域（占5/6宽度）
- **底部面板**: 日志窗口，显示运行日志和系统信息

## 节点类型

### 基础节点
- **开始节点**: 工作流的起始点
- **结束节点**: 工作流的结束点
- **数据处理节点**: 执行数据处理操作
- **条件判断节点**: 根据条件进行分支
- **循环节点**: 实现循环逻辑
- **文件操作节点**: 执行文件读写操作
- **网络请求节点**: 发送HTTP请求
- **数据库节点**: 执行数据库操作

## 使用方法

1. 运行主程序:
   ```bash
   python main.py
   ```

2. 从左侧节点库拖拽节点到右侧编辑器
3. 连接节点形成工作流
4. 使用菜单或工具栏保存工作流
5. 点击运行按钮执行工作流

## 快捷键

- `Ctrl+N`: 新建工作流
- `Ctrl+O`: 打开工作流
- `Ctrl+S`: 保存工作流
- `Ctrl+Q`: 退出程序
- `Ctrl+Z`: 撤销
- `Ctrl+Y`: 重做
- `Ctrl+C`: 复制
- `Ctrl+V`: 粘贴
- `F5`: 执行工作流
- `F6`: 验证工作流
